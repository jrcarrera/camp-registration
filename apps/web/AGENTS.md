# Web Engineering Guide

- The browser uses the documented API and generated contracts. Do not import
  database, server domain, or provider modules.
- Build responsive operational interfaces for repeated work, not marketing pages.
- Use semantic HTML, keyboard-accessible controls, visible focus, and descriptive
  labels.
- Use Lucide icons where a standard icon exists and add accessible names or
  tooltips to icon-only controls.
- Keep page state stable across loading, empty, error, and populated states.
- Do not expose Restricted data in URLs, browser logs, analytics, or cached public
  responses.
- Verify desktop and mobile screenshots after meaningful visual changes.
