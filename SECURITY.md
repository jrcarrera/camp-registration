# Security Policy

## Reporting a Vulnerability

Do not report vulnerabilities or sensitive data in a public issue.

Use GitHub's private security advisory workflow for this repository:

<https://github.com/jrcarrera/camp-registration/security/advisories/new>

Include affected versions, reproduction steps, impact, and suggested mitigation
when available. Do not include real camper, family, health, payment, or credential
data in a report.

## Scope

Security-sensitive areas include authorization, tenant isolation, health data,
uploads, payments, passwordless authentication, audit records, exports,
registration capacity, and provider webhooks.

No regulatory certification or compliance status is claimed by the project.
